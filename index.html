Cheat Sheet

Git & GitHub Commands

Quick, clean reference for daily Git work: setup, branches, remote, stash, tags, logs, cleaning, and more.

What’s the difference?

Git is a version control system for tracking changes locally.

GitHub hosts your Git repositories online for collaboration.

Tip: Use SSH for smoother pushes/pulls once your key is set.

Basics

Start, clone, check status, stage, commit.

Clone a repository
Copies a remote repo to your local machine (HTTPS or SSH).
git clone <URL>
Initialize a new repository
Creates a new empty Git repository in the current folder.
git init
Check current status
Shows the state of working directory and staging area.
git status
Stage files
Add file(s) to staging. Use . to stage everything.
git add <file-name>
git add .
Commit staged changes
Saves staged changes to local repository history.
git commit -m "message"

Config & Aliases

View or edit Git configuration and shortcuts.

Show config
Lists config values.
git config --list
git config -l
Show config with file origins
Shows where each config value comes from.
git config -l --show-origin
Set global config
Applies to all repositories for your user.
git config --global <name> "value"
Unset global config
Removes a global config key.
git config --global --unset <name>
Edit global config file
Opens the global config file in your editor.
git config --global --edit
Create a global alias
Example: git st becomes git status.
git config --global alias.<alias> "<command>"
List aliases
Shows all global aliases.
git config --global --get-regexp alias
Remove an alias
Unsets a global alias.
git config --global --unset alias.<alias>

SSH (GitHub)

Generate and test SSH connection.

Generate SSH key
Creates a 4096-bit RSA key with a comment (usually your email).
ssh-keygen -t rsa -b 4096 -C "email@example.com"
Show public key
Display your public key file content.
cat ~/.ssh/id_rsa.pub
Test GitHub SSH connection
Verifies your SSH auth to GitHub.
ssh -T git@github.com

Branches

Create, switch, rename, merge, delete.

List branches
Shows all local branches.
git branch
Create a branch
Creates a new local branch.
git branch <branch-name>
Switch branches
Moves you to another branch.
git checkout <branch-name>
Create and switch
Creates a new branch and checks it out immediately.
git checkout -b <branch-name>
Rename current branch
Renames the currently checked-out branch.
git branch -m <new-branch-name>
Merge a branch
Merges the specified branch into your current branch.
git merge <branch-name>
Delete a branch (safe)
Deletes only if it’s fully merged.
git branch -d <branch-name>
Delete a branch (force)
Forces deletion even with unmerged changes.
git branch -D <branch-name>

Remote

Origins, push, pull, remote branches.

Show remotes
Lists remote names and their URLs.
git remote -v
Add remote origin (SSH)
Adds a remote named origin using SSH URL.
git remote add origin git@github.com:<username>/<repo-name>.git
List remote branches
Shows remote branches (after fetch).
git branch -r
Push a branch
Uploads your local branch changes to the remote.
git push <remote-name> <branch-name>
Pull a branch
Fetches and merges remote changes into your local branch.
git pull <remote-name> <branch-name>
Force push (danger)
Overwrites remote history. Use only when you know what you’re doing.
git push <remote-name> <branch-name> --force

Stash

Temporarily save local changes and restore them later.

Stash changes
Stashes tracked changes and cleans the working directory.
git stash
Stash with message
Saves stash with a custom label.
git stash save "message"
List stashes
Shows saved stash entries.
git stash list
Apply latest stash (keep it)
Applies but does not remove it from stash list.
git stash apply
Pop latest stash (remove it)
Applies and removes the latest stash entry.
git stash pop
Pop a specific stash
Applies and removes a specific stash entry.
git stash pop stash@{<number>}
Show stash summary
Shows what changed in the stash.
git stash show
git stash show stash@{<number>}
Drop stash entries
Remove latest or a specific stash entry.
git stash drop
git stash drop stash@{<number>}
Clear all stashes
Deletes all stash entries.
git stash clear

Undo & Clean

Unstage, reset, clean untracked files, ignore files.

Unstage a file (classic)
Moves a file out of staging, keeps changes in working directory.
git reset HEAD <file-name>
Unstage a file (recommended)
Modern alternative to unstage.
git restore --staged <file-name>
Preview clean
Shows what would be removed (dry-run).
git clean -n
Remove untracked files
Deletes untracked files. Add -d to include directories.
git clean -f
git clean -fd
Hard reset to a commit (danger)
Discards changes after that commit in your working directory.
git reset --hard <commit-hash>
Soft reset to a commit
Moves HEAD but keeps changes staged.
git reset --soft <commit-hash>
Create .gitignore
Create and edit ignore rules.
touch .gitignore
code .gitignore
Force add ignored file
Stages a file even if it’s in .gitignore.
git add -f <file-name>

Logs

See commit history.

Show commit history
Displays commits with hashes, messages, authors, and dates.
git log

Tags

Create, list, push, and delete tags.

Create a lightweight tag
Points to the latest commit.
git tag <tag-name>
Create an annotated tag
Adds a message and metadata.
git tag -a <tag-name> -m "message"
List tags
Shows all tags (or with patterns).
git tag
git tag -l "v1.*"
Push a tag
Uploads a tag to the remote.
git push origin <tag-name>
Delete tag locally & remotely
Removes tag from local and remote.
git tag -d <tag-name>
git push origin --delete <tag-name>

Advanced

Cherry-pick commits from other branches.

Cherry-pick a commit
Apply a commit from another branch into current branch.
git cherry-pick <commit-hash>
Cherry-pick and add origin reference
Adds a line indicating the original commit hash.
git cherry-pick -x <commit-hash>
Cherry-pick and edit message
Opens editor so you can change the commit message.
git cherry-pick -e <commit-hash>

Concepts

Fork vs Clone.

Fork
Used by an external contributor who doesn’t have write access. It copies the repo to their GitHub account. After changes, they open a Pull Request for review. 
GitHub UI → Fork → Work → Pull Request
Clone
Copies a repository from GitHub to your local machine to work on it locally. 
git clone <URL>
